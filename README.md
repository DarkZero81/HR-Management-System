# HR Engine | نظام إدارة الموارد البشرية الذاتية

**نظام ويب متكامل لأتمتة عمليات الموارد البشرية، مبني على Laravel 13، يمنح كل مستخدم (موظف، مدير، إدارة) بوابته الخاصة لإنجاز معاملاته دون أي تدخل ورقي أو يدوي.**

---

## 📋 نظرة عامة

**HR Engine** هو نظام داخلي (Intranet System) يحوّل العمليات الإدارية اليومية للموارد البشرية — من الحضور والانصراف إلى الرواتب والإجازات والوثائق الرسمية — من معاملات ورقية بطيئة إلى بوابة رقمية موحّدة. صُمم النظام حول فكرة **الخدمة الذاتية (Self-Service)**: كل موظف يدير بياناته ويتابع مستحقاته بنفسه، وكل مدير يراجع ويوافق مباشرة من لوحته، بينما تحتفظ الإدارة العليا بسجل تدقيق كامل (Audit Trail) لكل عملية تغيير في النظام.

النظام مبني كتطبيق **Monolith Laravel** (Blade + Tailwind CSS) بدون الاعتماد على SPA منفصل، ما يجعله خفيفاً وسريع النشر على أي استضافة PHP تقليدية، مع بنية جاهزة للتوسّع مستقبلاً نحو واجهة API/موبايل منفصلة.

---

## 🎯 المشكلة التي يحلّها النظام

| قبل النظام | بعد النظام |
|---|---|
| طلبات إجازة عبر واتساب/ورق، بلا أرشفة | طلب رقمي موثّق بحالة (قيد المراجعة / موافق / مرفوض) مع مسار اعتماد واضح |
| حساب الرواتب يدوياً كل شهر | محرك رواتب (Payroll Engine) يحتسب الراتب الصافي تلقائياً من الحضور + البدلات + الخصومات |
| متابعة الحضور بدفاتر أو ملفات إكسل مبعثرة | ربط مباشر مع أجهزة البصمة وتسجيل التأخير/الإضافي آلياً |
| ضياع وثائق الموظفين (هوية، عقد، جواز سفر) | أرشفة رقمية مركزية لكل وثيقة مرتبطة بتاريخ انتهاء |
| لا يوجد سجل لمن عدّل ماذا | Audit Log كامل لكل عملية إضافة/تعديل/حذف على مستوى النظام |

## 👥 أدوار المستخدمين وصلاحياتهم

النظام مبني على نظام صلاحيات قائم على الأدوار (**Role-Based Access Control**) بثلاثة مستويات:

### 🧑‍💼 الموظف (Employee) — بوابة الخدمة الذاتية
- عرض بطاقة الأداء الشخصي والمؤشرات الفردية
- متابعة سجل الحضور والانصراف الشهري (أوقات الدخول، دقائق التأخير، ساعات الإضافي)
- استخراج قسيمة الراتب التفصيلية (Payslip) بصيغة **PDF** بدعم كامل للغة العربية
- تقديم طلبات (إجازة، إذن، سلفة) ومتابعة حالتها لحظياً
- رفع الوثائق الرسمية (هوية، عقد، جواز سفر، شهادة صحية) وربطها بملفه
- تسجيل الحضور والانصراف بنفسه من لوحة الخدمة الذاتية
- تصدير سجل الحضور والطلبات بصيغة **CSV**

### 👔 المدير المباشر / رئيس القسم (Manager)
- لوحة تحكم لحظية لعدد الموظفين المداومين بالقسم
- مراجعة واعتماد/رفض طلبات فريقه مباشرة
- إدارة ملفات الموظفين في قسمه
- متابعة سجل الحضور لفريقه

### 🛡️ الإدارة العليا (Admin)
- كل صلاحيات النظام بدون قيود
- إدارة كاملة لملفات الموظفين والأقسام والورديات والعطل الرسمية
- إدارة أجهزة البصمة والتحقق من حالة اتصالها
- مراجعة **سجل التدقيق الكامل (Audit Log)**: من أضاف/عدّل/حذف، وماذا، ومتى
- توليد أوامر صرف الرواتب الشهرية للجميع دفعة واحدة
- التقارير التشغيلية والمالية الإجمالية
- إرسال رسائل SMS/WhatsApp جماعية للموظفين
- إدارة إعدادات النظام

## ✨ الميزات الأساسية

- 🔐 **مصادقة OTP كاملة** — تسجيل دخول عبر رمز تحقق 6 أرقام يصل للبريد الإلكتروني، مع حماية ضد المحاولات الفاشلة والقفل المؤقت
- 🕐 **محرك حضور وانصراف** مرتبط بأجهزة بصمة (Device Integration)، مع احتساب آلي لدقائق التأخير والإضافي حسب وردية كل موظف
- 💰 **محرك رواتب (Payroll Engine)** يحسب صافي الراتب تلقائياً = الراتب الأساسي + البدلات − الخصومات، اعتماداً على الحضور والوقوعات المعتمدة
- 📄 **تصدير قسيمة راتب PDF بدعم عربي كامل** (عبر mPDF، مع ربط صحيح للحروف واتجاه RTL سليم)
- 📝 **دورة طلبات متكاملة (HR Transactions)**: تقديم → مراجعة المدير → اعتماد الإدارة → خصم/تسوية تلقائي لرصيد الإجازة
- 📁 **أرشفة وثائق رقمية** لكل موظف مع تتبع تاريخ الانتهاء وإشعارات تلقائية
- 🏢 **إدارة كاملة** للأقسام، الورديات، العطل الرسمية، وأجهزة البصمة
- 🧾 **سجل تدقيق (Audit Log)** يسجّل كل عملية Create/Update/Delete مع القيم القديمة والجديدة، لمن قام بها ومتى
- 🌓 **وضع داكن / فاتح (Dark/Light Theme)** قابل للتبديل ومحفوظ لكل مستخدم
- 📱 **تصميم متجاوب بالكامل (Responsive)** مع قائمة جانبية مخصصة للهواتف
- 🖨️ **تقارير تشغيلية ومالية** قابلة للتصدير (PDF, CSV)
- 🔔 **Modal تفاصيل الطلب** — عرض تفاصيل أي طلب بنافذة منبثقة بدون مغادرة الصفحة
- ⚡ **تحسينات الأداء** — Cache للبيانات الثابتة، Pagination للجداول الكبيرة، SoftDeletes لحماية البيانات
- 🗂️ **ورديات ليلية مدعومة** — إمكانية إنشاء ورديات تمتد عبر منتصف الليل مثل Evening Shift و Night Shift
- 📊 **بوابة الموظف (My Area)** — تسجيل حضور/انصراف شخصي، عرض سجل الحضور، تصدير CSV، رفع وثائق شخصية
- 📱 **نظام إشعارات متكامل** — إشعارات بريد إلكتروني (OTP, ترحيب، انتهاء وثائق) وقنوات SMS/WhatsApp قابلة للتوسع
- 🔒 **حماية الحسابات** — Middleware يمنع دخول الحسابات المعطلة تلقائياً
- 👤 **أفاتار المستخدم** — إمكانية رفع صورة شخصية للمستخدم
- ⚙️ **إعدادات النظام** — واجهة لتخزين واسترجاع إعدادات النظام بشكل مركزي
- 🛡️ **Observers و Cache Invalidation** — حذف تلقائي لـ OTP Codes القديمة وتحديث الكاش عند تعديل الورديات
- 🔄 **API RESTful موحّد** — استجابات API موحّدة عبر ApiResponseTrait مع Pagination و Auth Sanctum

## 🗂️ بنية قاعدة البيانات (ERD Summary)

النظام مبني على 15 جدولاً رئيسياً بعلاقات مترابطة بالكامل:

```
roles_permissions ─┬─< users ─┬─< employees ─┬─< documents
                      │          │              ├─< attendance_logs >─ attendance_devices
                      │          │              ├─< hr_transactions
                      │          └─< audit_logs  └─< payroll_orders
                      │
                  shifts ────────< employees
system_settings ───< (global)
otp_codes ─────────< users
```

**أهم الجداول:**
- `users` — حسابات المستخدمين مع is_active و avatar
- `employees` — الملف الأساسي لكل موظف (البيانات الشخصية، الراتب الأساسي، رصيد الإجازة، الوردية)
- `attendance_logs` — سجل يومي للحضور مربوط بجهاز البصمة، بحالة (حاضر/متأخر/غائب/إجازة)
- `hr_transactions` — الطلبات (إجازة، إذن، سلفة، ترقية) بدورة اعتماد كاملة
- `payroll_orders` — أوامر صرف الرواتب الشهرية (البدلات، الخصومات، الصافي)
- `audit_logs` — سجل تدقيق شامل لكل تغيير بالنظام
- `otp_codes` — رموز OTP لتسجيل الدخول مع Hash للرمز وتتبع المحاولات الفاشلة
- `system_settings` — إعدادات النظام المركزية
- `shifts` — الورديات مع دعم Overnight Shifts

## 🚀 التشغيل المحلي (Getting Started)

### المتطلبات
- PHP 8.3+
- Composer
- Node.js 20+ و npm
- MySQL / MariaDB

### خطوات التثبيت

```bash
# 1. استنساخ المشروع
git clone https://github.com/DarkZero81/HR-Management-System.git
cd HR-Management-System

# 2. تثبيت الحزم
composer install
npm install

# 3. إعداد ملف البيئة
cp .env.example .env
php artisan key:generate

# 4. إعداد قاعدة البيانات (بعد إنشاء قاعدة بيانات فارغة وتحديث بيانات الاتصال بملف .env)
php artisan migrate:fresh --seed

# 5. تشغيل المشروع (سيرفر + تصميم في نفس الوقت)
composer run dev
```

سيفتح المشروع تلقائياً على `http://127.0.0.1:8000`.
